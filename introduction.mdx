# Welcome to AgentDoor

AgentDoor is **the pre-auth layer for the agentic internet**. It lets AI agents register, authenticate, and pay for your API programmatically in under 500ms — with zero browser automation.

<Note>
  Make your website agent-ready in 3 lines of code. Let AI agents discover, register, and use your API without opening a browser or filling forms.
</Note>

## The Problem

AI agents can pay for APIs (x402), discover tools (MCP), and talk to each other (A2A). But they still **can't sign up for your product** without opening a browser, filling out forms, and solving CAPTCHAs.

The pre-auth funnel — registration → credential issuance → billing setup — was designed entirely for humans. Agents resort to slow (30–60s), fragile browser automation that breaks on every UI change.

## How It Works

AgentDoor implements a lightweight protocol for headless agent onboarding:

<Steps>
  <Step title="Discovery">
    Agent fetches `/.well-known/agentdoor.json` to discover available scopes, pricing, and endpoints.
    
    **~50ms**
  </Step>
  
  <Step title="Registration">
    Agent sends public key and requested scopes to `/agentdoor/register`. Server responds with a challenge nonce.
    
    **~100ms**
  </Step>
  
  <Step title="Challenge-Response">
    Agent signs the nonce with its private key and sends to `/agentdoor/register/verify`. Server issues API key + JWT.
    
    **~200ms**
  </Step>
  
  <Step title="Authenticated Requests">
    Agent makes requests with `Authorization: Bearer` header. Private key never transmitted.
    
    **Ongoing**
  </Step>
</Steps>

**Total time: < 500ms** (Compare: browser automation = 30–60s)

## Why AgentDoor?

### Speed

| Method | Time | Round-trips |
|--------|------|-------------|
| Browser automation | 30-60s | Many |
| OAuth 2.1 | 3-5s | 5+ |
| **AgentDoor** | **&lt;500ms** | **2** |

### Security

<Tip>
  Private keys never leave the agent. Every token refresh requires signing a fresh challenge with the Ed25519 private key — more secure than typical refresh tokens.
</Tip>

### Developer Experience

**For SaaS owners:** 3 lines to make your API agent-ready

**For agent developers:** 1 line to connect to any AgentDoor-enabled service

## Technical Highlights

- **< 5ms auth verification** — Ed25519 signature verification is extremely fast
- **< 2ms middleware overhead** — Minimal impact on request latency
- **< 50KB SDK** — Lightweight for constrained agent environments
- **Pluggable storage** — In-memory (dev) → SQLite → Postgres
- **Node.js >= 18** — Native crypto, ES modules, async/await

## Cross-Protocol Auto-Generation

One AgentDoor integration auto-generates companion protocol files:

| File | Protocol | Status |
|------|----------|--------|
| `/.well-known/agentdoor.json` | AgentDoor | Primary |
| `/.well-known/agent-card.json` | A2A (Google) | Auto-generated |
| `/.well-known/oauth-authorization-server` | OAuth 2.1 | Optional |
